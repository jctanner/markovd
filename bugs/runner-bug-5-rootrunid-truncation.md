---
title: rootRunID() truncates markov-run-* IDs to 8 chars, breaking event processing
severity: critical
component: api/events
---

## Problem

`processEvent()` in `internal/api/events.go` calls `rootRunID()` and `forkID()` to split compound sub-workflow run IDs. These functions assume the root run ID is always 8 hex characters (e.g., `97639fd9`). After the run ID mismatch fix (bug-4), run IDs are now `markov-run-XXXXXXXX` (20 chars). The truncation produces garbage:

| Function | Input | Expected | Actual |
|----------|-------|----------|--------|
| `rootRunID()` | `markov-run-a3ab59e4` | `markov-run-a3ab59e4` | `markov-r` |
| `forkID()` | `markov-run-a3ab59e4` | `""` | `n-a3ab59e4` |

This causes `UpsertRunFromEvent()` and `UpsertStep()` to write to run ID `markov-r` instead of `markov-run-a3ab59e4`. The actual run record stays stuck at "running" with no steps.

## Evidence

```sql
-- The real run record (created by handleCreateRun, never updated by events):
SELECT id, run_id, status FROM runs WHERE run_id = 'markov-run-a3ab59e4';
 id |       run_id        | status
----+---------------------+---------
 17 | markov-run-a3ab59e4 | running

-- The phantom record (created by processEvent with truncated ID):
SELECT id, run_id, status FROM runs WHERE run_id = 'markov-r';
 id |  run_id  |  status
----+----------+-----------
 18 | markov-r | completed

-- Steps landed under the wrong run:
SELECT run_id, step_name, status FROM steps WHERE run_id = 'markov-r';
  run_id  | step_name |  status
----------+-----------+-----------
 markov-r | say-hello | completed
 markov-r | show-date | completed
 markov-r | combine   | completed
```

## Root Cause

```go
// events.go:61-66
func rootRunID(id string) string {
    if len(id) >= 8 {
        return id[:8]   // "markov-run-a3ab59e4" → "markov-r"
    }
    return id
}
```

This was written for the old 8-char hex run IDs generated by markov internally. The new `markov-run-XXXXXXXX` format from the k8s runner is longer and doesn't use the `{8-char-root}-{fork}` convention.

## Fix

`rootRunID()` and `forkID()` need to understand the new ID format. The compound sub-workflow IDs from markov now look like `markov-run-a3ab59e4-deploy_all-0-health_check`. The root is everything up to and including the hex suffix: `markov-run-a3ab59e4`.

Simplest approach: split on `-` and check if the ID starts with `markov-run-`, then treat the first three segments as the root:

```go
func rootRunID(id string) string {
    if strings.HasPrefix(id, "markov-run-") {
        // "markov-run-a3ab59e4-deploy_all-0" → "markov-run-a3ab59e4"
        parts := strings.SplitN(id, "-", 4) // ["markov", "run", "a3ab59e4", ...]
        if len(parts) >= 3 {
            return strings.Join(parts[:3], "-")
        }
        return id
    }
    // Legacy 8-char hex IDs
    if len(id) >= 8 {
        return id[:8]
    }
    return id
}
```

Similar update needed for `forkID()`.

## Resolution

Updated both `rootRunID()` and `forkID()` in `internal/api/events.go` to detect the `markov-run-` prefix and split on the first three segments (`markov-run-XXXXXXXX`) as the root. Legacy 8-char hex IDs still work via the existing fallback logic.

Added unit tests in `internal/api/events_test.go` covering both the new `markov-run-*` format and legacy 8-char format for both functions, including compound sub-workflow IDs and edge cases.
